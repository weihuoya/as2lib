Workflow in the Project:

  1. Aiming from an Technical Target

    The Team has an Idea for an Package and plans the creation:
    • Which Sense should a Package have?
    • How should it corresponded with other Packages?

  2. Definition of the Package

    The Team works out the clear Definition of the Package (some kind of raw scribble)
    • Which name should it use?
    • Integration from existing stuff?
    • Package rules?

  3. Definition from an Package-Master

    The Team defines the main Developer for this package.
    • He will be the person to rule about the future of this package!
    • He will be accountable for problems.
    • He will define future plans.

  4. Definition from Classes
